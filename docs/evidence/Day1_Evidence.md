Task 1
## 1. 为什么要开发校园轻集市？
1. **校园交易需求旺盛但缺少专属平台**
校内学生闲置物品（书籍、电子产品、衣物、运动器材）、二手代购、社团物资、自习占位、拼单代取等交易需求长期存在，现有微信群、QQ群、线下摆摊形式杂乱无规范。
2. **第三方平台不适配校园场景**
闲鱼、转转等通用二手平台面向全社会，存在校外人员混入、距离远、交易不安全、手续费高、无法精准匹配本校师生等问题，缺少校园专属身份校验。
3. **规范校园交易，降低管理风险**
线下摆摊占道、私发广告、私下交易被骗、物品纠纷频发，搭建校内集市平台可统一管控交易行为，方便学校规范管理校园流通。
4. **实训教学落地需求**
作为Web/软件开发实训项目，业务逻辑简单完整、场景贴近学生生活，便于完成需求分析、架构设计、编码开发、测试部署全流程，锻炼前后端、数据库综合开发能力。
5. **便利师生，节约资源**
促进物品循环利用，减少资源浪费，实现校内低成本便捷交易，满足学生省钱、置换、转手闲置的刚需。

## 2. 系统主要解决哪些实际问题？
1. **交易渠道零散混乱问题**
告别分散的班级群、朋友圈、临时地摊，统一线上交易入口，商品集中展示，查找更高效。
2. **身份混杂、交易安全隐患**
通过校园账号/学号实名认证，仅本校师生可发布商品、沟通交易，隔绝校外陌生人，减少诈骗、货不对板、线下人身安全风险。
3. **二手交易信息不对称**
学生可按分类检索闲置，支持价格、发布时间筛选，快速匹配供需，解决“想卖找不到买家、想买找不到货源”。
4. **线下摆摊管理难题**
无需占用校园公共场地摆摊，减轻校方场地管理、卫生、秩序维护压力。
5. **交易无留存、纠纷难追溯**
线上留存商品发布记录、聊天记录、交易记录，出现物品质量、价格纠纷可留存凭证，方便平台介入调解。
6. **多元校园需求无统一载体**
除二手买卖外，拼单、代取快递、社团物资租借、寻物招领等零散需求无集中发布渠道。
7. **学生闲置资源浪费**
搭建流转渠道，教材、家电、家具等可重复利用，降低学生生活开支。

- 买家私信卖家咨询商品、议价、约定自提时间地点

### （5）订单交易场景
- 下单预约、确认交易、线下自提确认完成订单
- 订单状态管理（待沟通、待自提、交易完成、取消订单）

### （6）便民附加场景
- 寻物启事、失物招领板块
- 校园拼单、代取快递、物品短期租借发布

### （7）后台管理场景（管理员）
- 用户管理：违规账号封禁、认证审核
- 商品管理：下架违规商品、审核违规发布内容
- 举报处理：处理虚假商品、欺诈举报、纠纷调解
- 数据统计：商品发布量、活跃用户、热门分类统计

# Task3 
## 一、阅读 src/main.ts 思考解答
1. **Vue 应用如何创建？**
通过 `createApp(App)` 基于根组件 App.vue 创建 Vue 应用实例，再链式调用 `.mount('#app')` 将应用挂载到页面 id 为 `app` 的 DOM 节点上，完成应用初始化渲染。
2. **Pinia 如何注册？**
先导入 `createPinia` 生成 Pinia 实例，再通过 `app.use(pinia)` 将 Pinia 插件安装到 Vue 应用，全局注册后所有页面、组件均可使用全局状态管理。
3. **Router 如何注册？**
导入提前导出的路由实例 router，执行 `app.use(router)` 将路由插件注册到 Vue 应用，让项目具备页面跳转、路由匹配能力。
4. **项目入口在哪里？**
`src/main.ts` 是整个 Vue 项目的**程序入口文件**，打包运行时最先执行该文件，完成 Vue 实例创建、插件注册、全局配置、挂载渲染整套初始化流程。

## 二、阅读 src/router/index.ts 思考解答
1. **Router 的作用是什么？**
- 定义项目所有页面路由规则：绑定访问路径与对应页面组件；
- 控制页面跳转、浏览器前进后退；
- 配置路由守卫，实现登录权限校验、页面访问拦截；
- 管理路由参数、页面传值，实现单页面无刷新切换。
2. **为什么首页能够显示？**
路由配置中设置了**根路径重定向**，访问 `/` 时自动跳转首页路由；同时配置首页路由项，绑定首页视图组件，路由匹配成功后渲染对应页面，因此打开项目默认展示首页。

## 三、阅读 src/App.vue 思考解答
### App.vue 在整个项目中的作用是什么？
1. 项目**顶层根组件**，所有页面、组件都嵌套在它内部；
2. 承载 `<router-view>` 路由出口，路由匹配到的页面会在此位置渲染；
3. 统一全局基础布局、全局公共样式、全局导航栏/侧边栏等通用框架；
4. 可存放全局通用逻辑、全局弹窗、全局监听等跨页面公共功能。

### 交互任务3：解释Router作用与首页渲染原理
1. **Prompt**
详细讲解Vue Router在Vue单页项目中的核心作用，结合src/router/index.ts路由配置文件，说明打开项目默认自动展示首页的底层实现原理。
2. **AI输出完整内容**
- Vue Router核心作用：
  ① 路径映射：定义浏览器访问路径与业务页面组件的绑定关系，实现单页面无刷新切换；
  ② 页面跳转：提供编程式、声明式两种跳转方式，管理浏览器前进、后退历史记录；
  ③ 路由传参：支持页面间参数传递，实现商品详情、订单详情等页面数据携带；
  ④ 权限管控：编写全局/局部路由守卫，校验登录状态，未登录用户禁止访问发布、个人中心等页面；
  ⑤ 渲染出口：配合App.vue中的`<router-view>`标签，自动渲染路由匹配到的页面组件。
- 首页默认渲染原理：
  路由配置文件包含两项关键配置，共同实现打开项目直接展示首页：
  ① 根路径重定向配置：`{ path: '/', redirect: '/home' }`，当访问项目根域名时，自动强制跳转至首页路由地址`/home`；
  ② 首页路由规则：单独配置`path: '/home'`，绑定首页视图组件，路由匹配成功后，首页组件会渲染至App.vue的`<router-view>`路由出口内，因此打开项目默认展示首页。
3. **个人理解**
Router是实现多页面切换的核心依赖，没有路由则项目只能展示单一页面。重定向配置是首页自动弹出的核心逻辑，所有路由匹配到的页面，最终都会在根组件App.vue的路由出口中渲染，同时路由守卫可以实现校园集市的登录权限拦截，符合校园交易场景安全需求。
4. **本次交互分段结论**
Vue Router管控项目全部页面访问逻辑，依靠根路径重定向配置实现首页默认展示；同时可通过路由守卫实现登录校验，适配校园集市权限管控需求。
